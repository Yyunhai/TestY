## 为什么需要人机验证

当前 TestY 项目已有的安全措施：

- 登录失败 5 次后锁定账户 30 分钟（`AuthService`）
- Session 认证拦截器（`SessionAuthInterceptor`）

**缺失的防护：**
- 注册接口无防机器人 → 可被批量注册垃圾账号
- 登录接口无防暴力破解（仅靠账户锁定，攻击者可轮换用户名）
- 密码重置接口无验证 → 可被枚举用户信息

## 推荐方案：Cloudflare Turnstile（免费）

> Cloudflare Turnstile 是免费的、对用户无感的验证服务，不需要用户做任何操作，国内网络可直接访问。

### 第 1 步：注册并获取密钥

1. 访问 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 注册/登录后，进入 **Turnstile** 页面
3. 点击 **Add Widget**，填写：
   - Widget name: `TestY`
   - Domain: 你的域名（如 `testy.example.com`，本地开发填 `localhost`）
   - Mode: **Managed**（推荐，自动判断是否需要交互）
4. 创建后获得两个密钥：
   - **Site Key**（前端用，公开的）
   - **Secret Key**（后端用，保密的）

### 第 2 步：前端集成

#### 2.1 在 `index.html` 中引入 Turnstile JS

编辑 `frontend/web/public/index.html`，在 `<head>` 中添加：

```html
<script src="https://challenges.cloudflare.com/turnstile/v0/api.js" async defer></script>
```

#### 2.2 创建 Turnstile 组件

新建 `frontend/web/src/components/TurnstileCaptcha.vue`：

```vue
<template>
  <div ref="turnstileContainer" class="turnstile-wrapper"></div>
</template>

<script>
export default {
  name: "TurnstileCaptcha",
  props: {
    siteKey: { type: String, required: true }
  },
  emits: ["verified", "expired", "error"],
  data() {
    return { widgetId: null };
  },
  mounted() {
    this.renderWidget();
  },
  beforeUnmount() {
    if (this.widgetId !== null && window.turnstile) {
      window.turnstile.remove(this.widgetId);
    }
  },
  methods: {
    renderWidget() {
      if (!window.turnstile) {
        // Turnstile JS 尚未加载，等待后重试
        const timer = setInterval(() => {
          if (window.turnstile) {
            clearInterval(timer);
            this.renderWidget();
          }
        }, 200);
        return;
      }
      this.widgetId = window.turnstile.render(this.$refs.turnstileContainer, {
        sitekey: this.siteKey,
        callback: (token) => this.$emit("verified", token),
        "expired-callback": () => this.$emit("expired"),
        "error-callback": () => this.$emit("error")
      });
    },
    reset() {
      if (this.widgetId !== null && window.turnstile) {
        window.turnstile.reset(this.widgetId);
      }
    }
  }
};
</script>

<style scoped>
.turnstile-wrapper {
  display: flex;
  justify-content: center;
  margin: 8px 0;
}
</style>
```

#### 2.3 在 AuthPage 中使用

修改 `frontend/web/src/components/AuthPage.vue`，在提交按钮上方插入验证码：

```vue
<template>
  <!-- ... 原有内容 ... -->

  <form class="auth-form" @submit.prevent="handleSubmit">
    <!-- ... 原有表单字段 ... -->

    <!-- 👇 添加人机验证（在 submit 按钮之前） -->
    <TurnstileCaptcha
      ref="captcha"
      site-key="你的_SITE_KEY"
      @verified="onCaptchaVerified"
      @expired="onCaptchaExpired"
      @error="onCaptchaError"
    />

    <button class="auth-submit" type="submit" :disabled="busy || !captchaToken">
      <!-- ... -->
    </button>
  </form>
</template>

<script>
import TurnstileCaptcha from "./TurnstileCaptcha.vue";

export default {
  name: "AuthPage",
  components: { TurnstileCaptcha },
  // ... 原有 props, emits ...
  data() {
    return {
      showPassword: false,
      captchaToken: ""  // 👈 新增
    };
  },
  methods: {
    onCaptchaVerified(token) {
      this.captchaToken = token;
    },
    onCaptchaExpired() {
      this.captchaToken = "";
    },
    onCaptchaError() {
      this.captchaToken = "";
    },
    handleSubmit() {
      if (!this.captchaToken) return;
      this.$emit("submit", this.captchaToken);  // 👈 将 token 传给父组件
    }
  }
};
</script>
```

#### 2.4 在 App.vue 中传递 token

修改 `frontend/web/src/App.vue` 的 `submitAuth` 方法，将验证码 token 一并发送：

```js
async submitAuth(captchaToken) {
  this.authBusy = true;
  this.clearMessage();
  try {
    if (this.authMode === "login") {
      const body = { ...this.loginForm, captchaToken };  // 👈 附加 token
      const result = await this.apiRequest("/api/auth/login", { method: "POST", body });
      // ... 其余不变
    } else if (this.authMode === "register") {
      const body = { ...this.registerForm, captchaToken };  // 👈 附加 token
      await this.apiRequest("/api/auth/register", { method: "POST", body });
      // ... 其余不变
    } else {
      const body = { ...this.resetForm, captchaToken };  // 👈 附加 token
      await this.apiRequest("/api/auth/reset-password", { method: "POST", body });
      // ... 其余不变
    }
  } catch (error) {
    this.setMessage(error.message || "请求失败，请稍后再试。", "error");
  } finally {
    this.authBusy = false;
  }
}
```

### 第 3 步：后端集成

#### 3.2 创建验证码验证服务

新建 `backend/modules/service/CaptchaService.java`：

```java
package com.ysalu.service;

import com.fasterxml.jackson.annotation.JsonProperty;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Service;
import org.springframework.web.client.RestTemplate;

import java.util.Collections;
import java.util.List;

@Service
public class CaptchaService {

    private static final Logger log = LoggerFactory.getLogger(CaptchaService.class);
    private static final String VERIFY_URL = "https://challenges.cloudflare.com/turnstile/v0/siteverify";

    @Value("${TESTY_TURNSTILE_SECRET_KEY:}")
    private String secretKey;

    @Value("${TESTY_CAPTCHA_ENABLED:false}")
    private boolean captchaEnabled;

    /**
     * 验证 Turnstile token。
     * 如果验证码功能未开启，直接返回 true。
     */
    public boolean verifyTurnstile(String token, String remoteIp) {
        if (!captchaEnabled) {
            return true;  // 未开启则跳过验证
        }
        if (token == null || token.trim().isEmpty()) {
            return false;
        }

        try {
            RestTemplate restTemplate = new RestTemplate();

            // 构建请求体
            java.util.Map<String, String> params = new java.util.LinkedHashMap<>();
            params.put("secret", secretKey);
            params.put("response", token);
            if (remoteIp != null) {
                params.put("remoteip", remoteIp);
            }

            TurnstileResponse response = restTemplate.postForObject(
                VERIFY_URL, params, TurnstileResponse.class
            );

            if (response == null) {
                log.warn("Turnstile verification returned null response");
                return false;
            }

            if (!response.isSuccess()) {
                log.warn("Turnstile verification failed: {}", response.getErrorCodes());
            }

            return response.isSuccess();
        } catch (Exception e) {
            log.error("Turnstile verification request failed", e);
            // 网络异常时可选择放行或拒绝，建议拒绝
            return false;
        }
    }

    // ---- 响应 DTO ----
    public static class TurnstileResponse {
        private boolean success;

        @JsonProperty("error-codes")
        private List<String> errorCodes;

        @JsonProperty("challenge_ts")
        private String challengeTs;

        private String hostname;

        public boolean isSuccess() { return success; }
        public void setSuccess(boolean success) { this.success = success; }
        public List<String> getErrorCodes() { return errorCodes != null ? errorCodes : Collections.emptyList(); }
        public void setErrorCodes(List<String> errorCodes) { this.errorCodes = errorCodes; }
        public String getChallengeTs() { return challengeTs; }
        public void setChallengeTs(String challengeTs) { this.challengeTs = challengeTs; }
        public String getHostname() { return hostname; }
        public void setHostname(String hostname) { this.hostname = hostname; }
    }
}
```

#### 3.3 在 AuthController 中调用验证

修改 `AuthController.java`，在登录、注册、重置密码三个接口中加入验证码校验：

```java
// 在 AuthController 构造函数中注入 CaptchaService
private final CaptchaService captchaService;

public AuthController(
        AuthService authService,
        SessionAuthorization sessionAuthorization,
        OperationLogService operationLogService,
        CaptchaService captchaService  // 👈 新增
) {
    this.authService = authService;
    this.sessionAuthorization = sessionAuthorization;
    this.operationLogService = operationLogService;
    this.captchaService = captchaService;
}

// 获取客户端 IP 的辅助方法
private String getClientIp(HttpServletRequest request) {
    String ip = request.getHeader("X-Forwarded-For");
    if (ip != null && !ip.isEmpty()) {
        ip = ip.split(",")[0].trim();  // 取第一个（最原始的客户端 IP）
    } else {
        ip = request.getRemoteAddr();
    }
    return ip;
}

// 登录接口
@PostMapping("/login")
public ResponseEntity<?> login(@RequestBody LoginRequest req, HttpServletRequest request) {
    // 👇 验证码校验
    if (!captchaService.verifyTurnstile(req.getCaptchaToken(), getClientIp(request))) {
        return ResponseEntity.badRequest().body(Collections.singletonMap("message", "人机验证失败，请重试。"));
    }
    // ... 原有登录逻辑
}

// 注册接口
@PostMapping("/register")
public ResponseEntity<?> register(@RequestBody RegisterRequest req, HttpServletRequest request) {
    // 👇 验证码校验
    if (!captchaService.verifyTurnstile(req.getCaptchaToken(), getClientIp(request))) {
        return ResponseEntity.badRequest().body(Collections.singletonMap("message", "人机验证失败，请重试。"));
    }
    // ... 原有注册逻辑
}

// 重置密码接口
@PostMapping("/reset-password")
public ResponseEntity<?> resetPassword(@RequestBody ResetPasswordRequest req, HttpServletRequest request) {
    // 👇 验证码校验
    if (!captchaService.verifyTurnstile(req.getCaptchaToken(), getClientIp(request))) {
        return ResponseEntity.badRequest().body(Collections.singletonMap("message", "人机验证失败，请重试。"));
    }
    // ... 原有重置密码逻辑
}
```

## 备选方案一：Google reCAPTCHA v3

> 如果你的用户主要在海外，或已有 Google 账号体系，reCAPTCHA v3 也是很好的选择。


```js
  grecaptcha.execute('你的_SITE_KEY', { action: 'login' }).then(function(token) {
    // 将 token 作为 captchaToken 字段发送到后端
  });
});
```

```java
// 验证 URL: https://www.google.com/recaptcha/api/siteverify
// 参数: secret, response, remoteip
```

## 备选方案二：自建数学验证码

> 适用于无法接入第三方服务的内网环境。安全性较低，但零外部依赖。

### 后端：生成验证码

```java
@RestController
@RequestMapping("/api/captcha")
public class CaptchaController {

    // 用 Session 存储答案
    @GetMapping("/math")
    public ResponseEntity<?> generateMathCaptcha(HttpServletRequest request) {
        int a = (int) (Math.random() * 20) + 1;
        int b = (int) (Math.random() * 20) + 1;
        String[] operators = {"+", "-", "×"};
        String op = operators[(int) (Math.random() * operators.length)];

        int answer;
        switch (op) {
            case "+": answer = a + b; break;
            case "-": answer = a - b; break;
            default:  answer = a * b; break;
        }

        // 存入 Session
        request.getSession().setAttribute("CAPTCHA_ANSWER", String.valueOf(answer));

        return ResponseEntity.ok(Map.of(
            "question", a + " " + op + " " + b + " = ?",
            "sessionId", request.getSession().getId()
        ));
    }
}
```

### 前端：展示并输入

```vue
<template>
  <div class="math-captcha">
    <span class="captcha-question">{{ captchaQuestion }}</span>
    <input
      v-model="captchaAnswer"
      class="form-input"
      placeholder="请输入计算结果"
      @input="$emit('verified', captchaAnswer)"
    />
  </div>
</template>

<script>
export default {
  name: "MathCaptcha",
  emits: ["verified"],
  data() {
    return {
      captchaQuestion: "",
      captchaAnswer: ""
    };
  },
  async mounted() {
    const res = await fetch("/api/captcha/math");
    const data = await res.json();
    this.captchaQuestion = data.question;
  }
};
</script>
```

### 后端验证

```java
// 在登录/注册接口中
String expected = (String) request.getSession().getAttribute("CAPTCHA_ANSWER");
if (!req.getCaptchaAnswer().equals(expected)) {
    return ResponseEntity.badRequest().body(Map.of("message", "验证码错误"));
}
request.getSession().removeAttribute("CAPTCHA_ANSWER");  // 一次性
```

### 安全提醒

- 自建验证码防护强度很低，只能阻挡最基础的脚本
- 建议仅作为无法接入第三方服务时的兜底方案
- 可配合 IP 限流提升安全性

## 需要保护的接口

以下是建议添加人机验证的公开接口（无需登录即可访问）：

| 接口 | 路径 | 建议 |
|------|------|------|
| 登录 | `POST /api/auth/login` | ✅ 必须添加 |
| 注册 | `POST /api/auth/register` | ✅ 必须添加 |
| 重置密码 | `POST /api/auth/reset-password` | ✅ 必须添加 |
| 登出 | `POST /api/auth/logout` | ❌ 不需要 |
| 已认证接口 | `/api/**`（其他） | ❌ 不需要（已有 Session 认证） |
